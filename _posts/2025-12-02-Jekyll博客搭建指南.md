本文将详细介绍如何使用 Jekyll 和 GitHub Pages 搭建个人博客，从环境配置到部署上线的完整流程。

## 什么是 Jekyll？

Jekyll 是一个简单的、博客-aware 的静态网站生成器。它非常适合个人博客、项目文档等场景。主要特点包括：

- **静态生成**：生成纯 HTML 文件，加载速度快
- **Markdown 支持**：使用 Markdown 编写内容
- **模板系统**：支持布局和包含文件
- **插件生态**：丰富的插件支持
- **GitHub 集成**：与 GitHub Pages 完美配合

## 环境准备

### 1. 安装 Ruby

Jekyll 是基于 Ruby 的，首先需要安装 Ruby 环境：

**Windows:**
1. 下载 [RubyInstaller](https://rubyinstaller.org/)
2. 安装时勾选 "Add Ruby executables to your PATH"
3. 安装完成后验证：`ruby -v`

**macOS:**
```bash
# 使用 Homebrew 安装
brew install ruby
```

**Linux:**
```bash
# Ubuntu/Debian
sudo apt-get install ruby ruby-dev build-essential
```

### 2. 安装 Jekyll

```bash
gem install jekyll bundler
```

验证安装：
```bash
jekyll -v
```

## 创建博客

### 1. 新建 Jekyll 站点

```bash
jekyll new my-blog
cd my-blog
```

### 2. 本地预览

```bash
bundle install
bundle exec jekyll serve
```

访问 `http://localhost:4000` 查看效果。

## 目录结构

```
my-blog/
├── _config.yml          # 配置文件
├── _includes/           # 可复用的组件
├── _layouts/            # 页面模板
├── _posts/              # 博客文章
├── _sass/               # Sass 样式文件
├── assets/              # 静态资源
├── _site/               # 生成的静态文件
└── index.md             # 首页
```

## 配置文件

`_config.yml` 是 Jekyll 的主要配置文件：

```yaml
# Site settings
title: "我的个人博客"
description: "记录技术学习与生活感悟"
baseurl: ""
url: "https://yourusername.github.io"
author: "Your Name"
email: "your-email@example.com"

# Build settings
markdown: kramdown
highlighter: rouge
theme: minima
plugins:
  - jekyll-feed
  - jekyll-sitemap
  - jekyll-seo-tag

# Pagination
paginate: 5
paginate_path: "/page:num/"
```

## 写作内容

### 创建文章

在 `_posts` 目录下创建 Markdown 文件，文件名格式：`YYYY-MM-DD-title.md`

```markdown
---
layout: post
title: "文章标题"
date: 2025-12-03 10:00:00 +0800
categories: [技术]
tags: [Jekyll, 教程]
---

这里是文章内容，支持 Markdown 语法。

## 标题

- 列表项
- 列表项

**加粗** 和 *斜体*
```

### Front Matter

每篇文章开头的 YAML 配置称为 Front Matter：

```yaml
---
layout: post           # 使用的布局
title: "文章标题"       # 文章标题
date: 2025-12-03       # 发布日期
categories: [技术]      # 分类
tags: [Jekyll]        # 标签
excerpt: "文章摘要"    # 自定义摘要
---
```

## 自定义主题

### 创建布局

在 `_layouts` 目录下创建模板文件：

```html
<!-- _layouts/default.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{{ page.title }} - {{ site.title }}</title>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
</head>
<body>
    {% include header.html %}
    
    <main>
        {{ content }}
    </main>
    
    {% include footer.html %}
</body>
</html>
```

### 添加样式

创建 `assets/css/style.scss`：

```scss
---
---

@import "minima";

/* 自定义样式 */
body {
    font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
}

.post-title {
    color: #333;
    font-size: 2rem;
}
```

## 部署到 GitHub Pages

### 1. 创建 GitHub 仓库

1. 登录 GitHub
2. 创建新仓库，命名为 `yourusername.github.io`
3. 设置为公开仓库

### 2. 推送代码

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/yourusername.github.io.git
git push -u origin main
```

### 3. 启用 GitHub Pages

1. 进入仓库 Settings
2. 找到 Pages 设置
3. 选择 Source 为 Deploy from a branch
4. 选择 main 分支和 / (root) 目录
5. 保存并等待部署

### 4. 访问博客

部署完成后，访问 `https://yourusername.github.io` 查看你的博客。

## 常见问题

### 1. 样式不生效

- 检查 baseurl 配置是否正确
- 确保 CSS 文件路径正确
- 检查 Jekyll 主题是否正确安装

### 2. 中文乱码

- 确保文件使用 UTF-8 编码
- 在 `_config.yml` 中添加：
  ```yaml
  encoding: utf-8
  ```

### 3. 本地预览正常，部署后异常

- 检查是否有插件在 GitHub Pages 上不受支持
- 查看 GitHub Actions 构建日志
- 确保文件名和路径没有特殊字符

## 进阶优化

### 1. 添加评论系统

可以使用 Disqus、Gitalk 等评论系统：

```html
<!-- 在 _layouts/post.html 中添加 -->
<div id="disqus_thread"></div>
<script>
    var disqus_config = function () {
        this.page.url = "{{ site.url }}{{ page.url }}";
        this.page.identifier = "{{ page.id }}";
    };
    (function() {
        var d = document, s = d.createElement('script');
        s.src = 'https://your-disqus-shortname.disqus.com/embed.js';
        s.setAttribute('data-timestamp', +new Date());
        (d.head || d.body).appendChild(s);
    })();
</script>
```

### 2. 添加搜索功能

使用 Jekyll 搜索插件或集成 Google 自定义搜索。

### 3. SEO 优化

- 使用 jekyll-seo-tag 插件
- 优化页面标题和描述
- 添加结构化数据
